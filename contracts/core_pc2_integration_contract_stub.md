## 1. Core Event Dispatch
To trigger marketing asset generation, the Core system will dispatch an HTTP POST request to the n8n entry webhook. The payload must conform to the Core-to-n8n event schema.

### Event Envelope
```json
{
  "contract_version": "e2e_dry_run_v0.1",
  "event_type": "creative_asset.requested",
  "request_id": "782f9d5e-ca87-439f-a89c-8eb139f4a088",
  "brand_id": "brand_demo_001",
  "campaign_id": "campaign_demo_001",
  "requested_by": "user_demo_001@example.com",
  "callback_url": "http://localhost:5678/mock-callback",
  "payload": {
    "asset_type": "image",
    "creative_brief": "A modern co-working space banner with glassmorphism effects"
  },
  "metadata": {}
}
```

## 3. Module Routing
The n8n router determines target stubs by inspecting `event_type`:
- `creative_asset.requested` → routes to ComfyUI local pipeline.
- `content_pack.requested` → routes to social content pack generator.
- `ads_pack.requested` → routes to ad creatives generator.
- `crm_followup.requested` → routes to CRM mockup drafter.
- `analytics_report.requested` → routes to analytics summary compiler.

---

## 4. Callback Preview
Upon completion of a module stub execution, the workflow normalizes the result into a callback preview schema. This preview contains:
- `module_status`: `"mock_completed"`, `"failed"`, or `"QUEUED"`.
- `output`: Generated asset paths or text variants.
- `errors`: List of validation or execution messages (if any).

---

## 5. Approval Decision
Core human-in-the-loop decisions are mapped inside the approval gate:
- `"approved"`: Promotes asset to final status (`completed_mock`).
- `"rejected"`: Stops workflow execution (`stopped_rejected`).
- `"needs_revision"`: Flags asset as requiring revision (`revision_required`).
- `"pending_approval"`: Holds asset in queue (`waiting_for_owner_approval`).

---

## 6. Error Callback
If any check fails (unsupported events, offline modules, or execution issues), n8n returns an error callback payload matching the N9 error policy:
- `error_type`: E.g., `unsupported_event_type`, `module_unavailable`, `unknown_error`.
- `error_code`: Standardized code (e.g. `ERR_ROUTER_UNSUPPORTED_EVENT`).
- `error_message`: Human-readable error details.
- `retryable`: Boolean indicating if retry can be scheduled.
